# LinkVault Resource Aggregator

LinkVault 是一个面向技术研究者、内容策展人和信息分析师的轻量级外链资源聚合与导航系统。该项目并非传统的爬虫框架或书签管理工具，而是一个结构化的外部链接索引仓库，专注于对分散在各类内容平台上的深度技术文章、行业报告和案例分析进行集中收录、分类标注与快速检索。

LinkVault 解决的核心问题是：技术人员在日常阅读中积累的大量有价值的外链（如移动端技术实践、行业趋势解读、架构设计案例等）缺乏统一的管理入口，往往散落在浏览器书签、即时通讯记录或笔记软件中，难以形成体系化的知识索引。本项目通过标准化的资源收录流程、分类目录结构和持续集成式的链接健康检查，帮助用户构建可维护、可共享的外链知识库。

项目本身不存储任何文章内容或媒体文件，仅保留链接元数据（标题、来源域名、收录批次、状态码等），确保轻量且合规。目标用户包括技术博客作者、开源社区维护者、企业培训部门以及任何需要系统化管理阅读资源的个人或团队。

## 功能概览

**批量链接收录** 支持按批次导入大量外链，自动提取域名和路径结构，生成标准化的资源条目，本次第 8 批次共收录 180 个链接。

**链接状态监控** 周期性对已收录链接发起 HTTP 请求，检测状态码变化，标记失效或重定向的链接，便于及时清理或更新。

**分类标签系统** 允许为每个链接添加自定义标签（如「移动端」「性能优化」「安全」「架构」等），支持多标签组合筛选。

**全文元数据提取** 对可访问的链接自动拉取页面标题、meta 描述和主要正文摘要，生成索引快照供离线检索。

**只读 API 接口** 提供 RESTful 风格的查询接口，支持按域名、标签、收录时间范围等条件检索已收录链接，返回 JSON 格式数据。

**Markdown 目录导出** 将当前索引库中的链接一键导出为结构化 Markdown 表格或列表，便于嵌入技术文档、周报或知识库页面。

**链接去重与冲突检测** 在收录新链接时自动比对已有数据库，发现重复 URL 或相似路径时发出警告，避免冗余条目。

**收录批次管理** 按批次记录链接来源、收录时间和操作人，支持回溯每一批次的原始数据，便于审计和协作。

## 应用场景

技术博客的内容策展。技术博主或媒体编辑在策划每周技术周刊时，需要从大量来源中筛选高质量文章。LinkVault 允许策展人预先将候选链接按批次导入，随后通过标签筛选和状态检查快速剔除不可用链接，最终导出整洁的 Markdown 链接列表直接用于发布。

企业内部技术知识库建设。许多研发团队会维护内部的技术文档站点，其中包含大量外部引用。使用 LinkVault 可以集中管理这些外部引用，定期检查链接可用性，避免文档中出现大量死链。同时，API 接口可与企业内部 Wiki 系统集成，实现链接的自动同步。

个人阅读工作流管理。技术从业者每天可能阅读数十篇技术文章，但事后回顾时往往难以找到特定主题的过往阅读材料。LinkVault 作为轻量级索引工具，可以配合浏览器的「发送到」功能或书签导出脚本，将日常阅读的链接统一汇总，并按主题打标签，构建个人化的技术阅读档案。

开源项目文档的外部引用维护。开源项目的 README 或官方文档中常引用外部资源（如设计规范、论文、参考实现等）。这些外部链接随着时间推移容易失效。LinkVault 的定期监控功能可以提前发现失效引用，提醒维护者及时更新文档中的链接，提升文档质量。

数据分析师的数据源登记。数据分析项目往往依赖多个外部数据源或 API 文档。LinkVault 可用于登记这些外部依赖的 URL，记录其更新频率和访问凭证要求，作为项目数据字典的补充部分，便于团队协作时快速了解数据来源。

## 快速开始

以下命令适用于 Linux 或 macOS 环境，Windows 用户可使用 WSL 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/your-organization/linkvault.git
cd linkvault

# 安装依赖（使用 pip 和虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化数据库并导入示例数据（包含第 8 批次资源）
python manage.py initdb
python manage.py import-batch --batch 8 --file ./data/batch_8.links

# 启动开发服务器
python manage.py runserver --port 8080
```

访问 http://localhost:8080 即可进入 LinkVault 的 Web 管理界面，默认管理员账号为 admin，密码在首次启动时输出于终端日志中。若需修改端口或绑定地址，可编辑 config.yaml 中的 server 配置段。

## 贡献指南

提交问题报告或功能请求。请在 GitHub Issues 中使用提供的模板填写，明确说明问题的复现步骤、预期行为和实际行为。对于功能请求，请详细描述使用场景和期望的 API 或界面变化。

代码贡献流程。首先 Fork 本仓库，在 dev 分支上创建功能分支（命名格式为 feature/简述或 fix/简述）。完成代码后运行 `pytest tests/` 确保所有测试通过，并使用 `black` 格式化代码。提交 Pull Request 到 dev 分支，PR 描述中需关联相关 Issue 编号。

文档改进。文档位于 docs/ 目录下，采用 Markdown 格式。若发现错别字、过期描述或示例错误，可直接提交 PR 修改。对 API 文档的变更需同步更新对应的代码注释和 OpenAPI 规范文件。

链接资源扩充。若希望向 LinkVault 的公共资源库贡献新的外链列表，请按 data/ 目录下的批次文件格式（每行一个 URL）提交，并附上该批次的主题描述和来源说明。项目维护者将审核链接的合法性和内容相关性。

本地开发环境搭建。使用 `pip install -r requirements-dev.txt` 安装开发依赖，并复制 `config.example.yaml` 为 `config.yaml` 进行本地配置。运行 `python manage.py initdb --force` 重置测试数据库。

## 常见问题

**问：LinkVault 是否存储目标链接的页面内容？是否会因为频繁请求导致目标网站压力过大？**

答：LinkVault 默认不保存页面全文或二进制内容，仅存储标题、meta 描述和响应状态码等元数据。健康检查模块采用可配置的并发数（默认 5 个并发请求）和间隔时间（默认 24 小时一次），且严格遵守 robots.txt 中的 Crawl-delay 指令。用户可在 config.yaml 中调整这些参数以降低对目标站点的访问频率。

**问：数据库是否可以迁移到 MySQL 或 PostgreSQL 生产环境？**

答：当前版本仅内置了 SQLite 驱动，适用于中小规模索引（建议不超过 5 万条链接）。对于更大规模或高并发读写的场景，项目提供了存储接口抽象层（internal/storage 包），用户可参考 sqlite_driver.py 实现对应的 MySQL 或 PostgreSQL 驱动，并在 config.yaml 中切换 driver 类型。社区已有人贡献了 PostgreSQL 适配的示例，可在 discussions 中查阅。

**问：导入批次时如果部分链接已存在，会怎么处理？**

答：导入命令默认采用「跳过重复」策略，即对已存在的 URL 不更新也不报错，仅输出警告日志到 stderr。若需强制覆盖已有条目的元数据（例如重新拉取标题），可在导入命令中添加 `--update-existing` 标志。去重判断依据为 URL 的标准化形式（去除末尾斜杠和空锚点后的字符串），详情参考 pkg/validator/dedup.py 中的实现。
