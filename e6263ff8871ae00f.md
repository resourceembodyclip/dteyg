# LinkVault Resource Aggregator

LinkVault 是一个面向技术研究者、内容策展人和开发者的轻量级外链资源汇总与导航系统。该项目旨在解决信息分散、优质内容难以追溯的问题，通过结构化的文章索引机制，将分散于各类技术博客、新闻站点与案例库中的深度内容统一归档，并提供可插拔的元数据抽取与检索接口。

项目采用静态资源聚合模式，不对原始内容做任何改写或重排，仅作为链接关系的整理与呈现层。目标用户包括需要维护技术周报的编辑、进行竞品分析的产研团队、以及希望系统化阅读特定领域文章的个人开发者。LinkVault 不依赖数据库，所有链接数据以纯文本格式存储，便于版本控制与协作编辑，同时提供基础的分类标签与全文检索支持，帮助用户在数百乃至数千条外链中快速定位目标条目。

## 功能概览

**结构化链接目录**：按照来源站点、主题领域或时间批次对链接进行多级分类，支持自定义标签体系，每条链接自动关联文章标识符与批次元数据。

**全量原样保留机制**：所有收录链接严格保持原始 URL 字符串，不做协议升级、域名规范化或路径改写，确保指向地址与提交时完全一致。

**轻量级元数据抽取**：通过可配置的解析规则，从链接路径中提取文章 ID、文件扩展名及潜在的分类线索，为后续筛选提供基础字段。

**纯静态资源索引**：无需后端服务或数据库支持，所有链接数据存储于单一 Markdown 文件或 JSON 清单中，可直接托管于任何静态服务器或 Git 仓库。

**多维度检索过滤**：基于文章编号、来源域名或路径关键词进行组合筛选，支持正则表达式匹配，满足复杂查询需求。

**批次管理能力**：内置批次编号与资源计数功能，当前批次为第 13/56 批，共收录 180 条链接，便于追踪资源引入进度与版本迭代。

**可扩展解析器接口**：提供抽象解析类，允许开发者针对不同 URL 模式编写自定义处理逻辑，实现标题猜测、日期提取或分类预测。

**协作友好型数据格式**：所有资源列表采用纯文本列表形式，每行一条记录，支持多人同时编辑，合并冲突概率低，适合通过 Pull Request 进行贡献。

## 应用场景

技术周报与月刊内容编排：编辑人员可将分散于不同技术博客、官方发行说明或社区讨论帖中的优质文章通过 LinkVault 统一收录，并按照本期主题批量导出为推荐阅读清单，极大缩短素材收集与整理时间。

竞品动态追踪与产品分析：产品经理与市场分析师可将竞争对手的版本发布公告、功能解读文章或用户案例研究集中归档，形成按时间线排列的外部情报库，便于周期性回顾与决策参考。

个人知识体系构建与主题阅读：开发者可将日常浏览中发现的碎片化技术文章按语言、框架或问题域分类存储，构建个人化的外链知识网络，避免重复搜索与遗忘，提升系统化学习效率。

离线存档与合规审查预备：法务或合规部门可将需要定期复审的外部引用链接统一纳入 LinkVault 管理，配合静态导出功能生成不可篡改的链接清单，满足审计与存档要求。

## 快速开始

以下命令序列演示了从仓库克隆、安装依赖到启动本地预览服务的完整过程，适用于 Linux 与 macOS 环境，Windows 用户建议使用 WSL 或 Git Bash。

```bash
git clone https://github.com/your-org/linkvault.git
cd linkvault
pip install -r requirements.txt
python -m linkvault serve --port 8080
```

执行上述命令后，本地服务将在 8080 端口启动，访问 http://localhost:8080 即可查看当前批次的资源列表。若需重新生成静态页面，可运行 `python -m linkvault build --output ./dist`，所有 HTML 与资源文件将输出至 dist 目录，可直接部署至 Nginx 或 S3 存储桶。

## 安装要求

| 依赖 | 必需 | 说明 |
|---|---|---|
| Python 3.9 或更高版本 | 是 | 核心运行环境，用于解析 URL、生成静态页及启动预览服务 |
| pip 22.0 以上 | 是 | Python 包管理工具，用于安装 requirements.txt 中声明的第三方库 |
| Git 2.25 以上 | 是 | 用于克隆仓库及后续版本管理，若只使用源码包可跳过 |

| 可选：watchdog | 否 | 用于开发模式下监听文件变动并自动重载服务，提升调试效率 |
| 可选：pytest 7.0 以上 | 否 | 仅当需要运行单元测试或贡献代码时安装 |
| 操作系统：Linux / macOS / Windows (WSL) | 是 | 核心功能在主流操作系统上均通过兼容性测试 |

## 项目结构

```
linkvault/
├── README.md                     # 项目总览与快速入门文档（即本文件）
├── LICENSE                       # MIT 许可证全文
├── requirements.txt              # Python 依赖列表（锁定主要版本号）
├── setup.py                      # 包安装与分发配置文件
├── config/                       # 全局配置目录
│   ├── settings.yaml             # 站点标题、描述、分页大小等核心配置
│   └── batch_list.json           # 所有批次的编号与资源计数映射表
├── data/                         # 资源数据存储目录（纯文本格式）
│   ├── batch_13_raw.txt          # 第 13 批原始链接列表（每行一条）
│   └── batch_13_metadata.json    # 第 13 批链接的附加元数据（如有）
├── src/                          # 核心源代码目录
│   ├── __init__.py
│   ├── parser.py                 # URL 解析与文章 ID 提取模块
│   ├── generator.py              # 静态 HTML / Markdown 渲染引擎
│   ├── server.py                 # 轻量级开发预览服务器（基于 http.server）
│   └── cli.py                    # 命令行入口，处理 build / serve / export 子命令
├── tests/                        # 单元测试与集成测试目录
│   ├── test_parser.py            # 针对 parser 模块的边界用例测试
│   └── test_generator.py         # 针对 generator 渲染结果的正确性校验
├── templates/                    # Jinja2 或原生 Python 模板文件（可选）
│   ├── index.html                # 资源列表主页面模板
│   └── detail.html               # 单条资源详情页模板（预留）
└── dist/                         # 构建输出目录（默认生成位置，可配置）
    ├── index.html                # 最终可部署的静态首页
    └── assets/                   # CSS / JavaScript 等静态资源
```

## 常见问题

Q: 为什么收录的链接地址包含 .sHtML 这样的非标准扩展名？是否会进行统一转换？

A: .sHtML 是源站点的实际路径格式，LinkVault 的核心原则是保持原始 URL 的原样输出，不做任何字符改写或协议升级。这样做是为了确保链接的精确可溯性，避免因自动规范化导致的访问失败。项目不会对扩展名的大小写或后缀进行任何转换操作。

Q: 如何迁移已存在的数千条链接到 LinkVault 系统？

A: 您可以将现有链接按每行一条的格式放入 data/ 目录下的批次文件中，文件名遵循 batch_序号_raw.txt 的模式。随后运行 python -m linkvault import --file 您的文件路径，系统会自动进行去重校验并生成对应的元数据占位。若需要批量修改链接前缀或域名，建议使用 sed 或 awk 等外部文本工具处理，LinkVault 自身不提供链接重写功能。

Q: 静态生成器能否输出 PDF 或 CSV 格式？

A: 当前版本默认输出 HTML 与 Markdown 两种格式。若需要 CSV 导出，可使用 python -m linkvault export --format csv 命令，该功能自 v1.2.0 起已内置于 cli.py 中。PDF 输出暂未支持，但您可以利用浏览器自带的打印功能将 HTML 页面另存为 PDF，或使用第三方工具如 weasyprint 进行转换。
