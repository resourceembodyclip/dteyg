# WapResourceAggregator

WapResourceAggregator 是一个面向移动端技术研究与内容聚合的开源项目，专注于收集、整理和归档来自移动互联网领域的公开技术文章、行业动态与案例分析。项目定位为技术资源的外链汇总与导航系统，服务于移动端开发者、技术决策者、产品经理以及行业分析师，帮助其快速定位特定主题下的高质量内容，减少信息检索成本，提升技术研究效率。

本项目不提供内容托管或镜像存储，仅作为结构化导航索引存在。所有链接指向原始来源，项目维护者不对第三方内容的可用性、准确性或合法性承担责任。项目通过批次化资源导入机制运行，当前为第 30/56 批，累计收录资源链接 180 条，涵盖移动 Web 实践、前端工程化、跨端方案、性能优化、安全策略等多个技术子领域。

## 功能概览

**结构化资源索引**：按照技术领域、内容类型与业务场景对资源链接进行多维度分类，支持按批次、按来源域、按内容特征进行快速筛选与定位。

**批量导入机制**：支持以文本列表形式批量导入 URL 资源，自动去重并生成批次标签，便于后续维护与增量更新。

**资源状态标记**：为每个资源链接提供状态标记字段，包括未读、已读、待归档、失效链接等状态，方便团队协作与阅读进度管理。

**标签与全文检索**：基于资源标题、描述、来源域及自定义标签构建轻量级检索能力，支持多关键词组合查询，帮助用户在大量链接中快速命中目标内容。

**可导出导航集**：支持将选定资源列表导出为 Markdown、HTML 或 JSON 格式，便于嵌入技术文档、周报或内部知识库系统。

**访问统计概览**：记录每个资源链接的点击次数与最后访问时间，辅助判断内容的热度与时效性价值。

## 应用场景

**移动端技术团队周报素材整理**：技术负责人或团队成员可在每周例会前使用本项目快速筛选当周新增的移动 Web 相关资源，从中提取高价值文章链接作为周报附件或讨论素材，减少重复搜索时间。

**技术选型与竞品分析**：产品经理与技术架构师在评估跨端框架、渲染方案或 Hybrid 架构时，可通过本项目检索过往批次中收录的案例分析、性能对比与最佳实践文章，获得第三方视角的参考依据。

**个人技术阅读清单管理**：移动端开发者可利用本项目的标签与状态标记功能维护个人阅读清单，将日常浏览中发现的零散文章统一归档，并按主题分类进行系统性阅读与笔记整理。

**开源文档外链引用校验**：开源项目维护者可将本项目作为外链引用来源池，在编写项目文档或技术博客时引用本项目收录的稳定链接，并通过本项目提供的失效标记功能定期校验引用链接的可用性。

## 快速开始

以下步骤适用于首次克隆本项目并启动本地导航服务的开发者。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/wap-resource-aggregator.git

# 进入项目目录
cd wap-resource-aggregator

# 安装依赖（基于 Node.js 18+ 与 npm）
npm install

# 启动本地开发服务器，默认监听端口 3000
npm run dev
```

启动成功后，访问 `http://localhost:3000` 即可查看资源列表与导航界面。如需导入新批次资源，请将 URL 列表存放于 `data/batches/batch_30.txt` 后执行 `npm run import`。

## 贡献指南

1.  Fork 本项目至个人账户，在本地创建功能分支（如 `feat/batch-31-import`），所有修改均应在分支上进行，禁止直接向主分支提交。

2.  新增资源批次时，请按照 `data/batches/batch_template.txt` 的格式准备纯文本 URL 列表，每行一个链接，并在提交前执行 `npm run validate` 检查所有链接是否可达且格式合规。

3.  若发现已有资源链接失效，请通过项目 Issue 模板提交「链接失效报告」，包含原始 URL、失效日期及可替代链接（如有）。维护者将定期合并此类修复。

4.  代码风格遵循 ESLint 配置（基于 Airbnb 规范），提交前请执行 `npm run lint` 与 `npm run test` 确保代码质量与测试通过率不低于 95%。

5.  提交信息请使用 Conventional Commits 格式（如 `feat: add batch 31 resources`、`fix: correct url validation regex`、`docs: update import guide`），便于自动生成变更日志。

## 常见问题

**问：导入批次资源时提示「链接格式不合法」，但我的 URL 在浏览器中能正常打开，是什么原因？**

答：项目内置的 URL 校验器要求每个链接必须包含协议头（http:// 或 https://）且不得包含换行符或多余空格。请检查您的批次文件是否包含 BOM 头、空行或不可见字符。若问题持续，可尝试使用 `dos2unix` 工具转换文件换行符后再执行导入。

**问：如何定期自动检查已收录链接的可用性？**

答：项目提供 `scripts/check-links.sh` 脚本，可配合系统定时任务（如 cron）每周执行。该脚本会遍历全部资源并记录 HTTP 状态码，将 4xx/5xx 及超时链接标记为「失效」并生成报告发送至配置的邮箱。具体配置方法请参阅 `docs/maintenance.md` 中的巡检章节。

**问：本项目是否可以部署为公开的在线导航站点？**

答：可以。项目内置了基于 Express 的轻量级 Web 服务，支持通过 `npm run build` 构建静态文件后部署至任何支持 Node.js 的云平台（如 Vercel、Railway 或自建服务器）。但请注意，本项目仅作为外链索引，不缓存任何第三方内容，部署前请确保遵守目标平台的合规政策与版权规定。
