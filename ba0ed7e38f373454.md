# ResourceBridge

ResourceBridge 是一个面向技术团队与个人开发者的外链资源归集与导航系统。该项目并非传统的爬虫或书签管理器，而是一个结构化的技术文献与参考资料索引库，专注于对分散在互联网各处的技术文章、API 文档、规范说明及解决方案进行系统性梳理与快速定位。ResourceBridge 适用于需要频繁查阅外部技术资料、维护项目依赖文档或构建团队知识库索引的场景，其核心价值在于将零散的 URL 转化为可检索、可分类、可版本控制的结构化数据资产。

## 功能概览

**结构化资源索引**：基于 Markdown 构建的层级化资源目录，支持按技术领域、问题类型或文档来源进行多维度分类。

**快速定位机制**：每个资源条目附带原始发布时间与内容摘要字段，支持通过 grep 或内置搜索脚本进行关键词匹配。

**版本化文档管理**：所有资源链接与描述信息纳入 Git 版本控制，支持回溯历史变更与团队协作审阅。

**自动化校验流水线**：集成 CI 脚本，周期性检测链接可用性与状态码变化，自动标记失效资源。

**离线阅读支持**：通过脚本批量拉取链接对应页面的纯文本内容，生成可供离线浏览的静态缓存。

**标签系统**：支持为每个资源条目附加自定义标签，便于按主题或项目维度进行过滤与聚合。

**导入导出兼容性**：资源列表支持导出为 JSON、CSV 及 HTML 书签文件格式，便于迁移至其他工具。

## 应用场景

技术团队内部知识库建设。团队可将日常开发中遇到的外部参考链接统一归集至 ResourceBridge，按项目或技术栈分类，新人入职时可快速了解项目所依赖的外部规范与常用解决方案。

个人开发者的阅读清单管理。开发者可将待阅读的技术文章、教程或 RFC 文档链接存入系统，利用标签标记阅读状态与优先级，形成个人技术成长路径。

开源项目的依赖文档索引。开源维护者可将项目所参考的第三方库文档、协议文本、上游依赖说明等资源统一整理，作为项目文档的补充附件，降低贡献者的入门门槛。

离线环境下的文档备援。对于内网开发环境或网络受限场景，ResourceBridge 的离线缓存功能可预先将关键资源页面保存为本地副本，确保在无外网连接时仍可查阅核心参考资料。

## 快速开始

以下命令序列适用于 Linux 及 macOS 环境，Windows 用户可使用 WSL 或 Git Bash 执行。

```bash
# 克隆仓库至本地
git clone https://github.com/example/resource-bridge.git
cd resource-bridge

# 安装依赖（基于 Python 3.9+）
pip install -r requirements.txt

# 初始化资源数据库并运行本地导航服务
python scripts/init_db.py --import data/resources.json
python scripts/serve.py --port 8080
```

执行完毕后，可通过浏览器访问 http://localhost:8080 查看资源导航界面。若需更新资源列表，可运行 `python scripts/sync.py` 从上游数据源拉取最新条目。

## 贡献指南

提交资源新增或更新。通过 Pull Request 向 `data/resources.json` 提交变更，需遵循既定 JSON Schema 格式，确保每个条目包含 `id`、`url`、`title`、`category`、`tags` 及 `last_verified` 字段。

运行本地校验。在提交前，执行 `python scripts/check_links.py --validate-only` 确保新增链接可访问且状态码为 2xx 或 3xx，避免引入失效资源。

更新文档与注释。若新增功能或修改脚本行为，需同步更新 `docs/` 目录下对应文档，并在代码中添加充分的注释说明。

编写或更新测试用例。对于脚本逻辑的改动，应在 `tests/` 目录下补充相应的单元测试，执行 `pytest` 确保全部测试通过。

提交 PR 描述规范。Pull Request 标题应简明概括变更内容，正文需说明变更动机、影响范围及测试结果，关联相关 Issue（如有）。

## 常见问题

Q: 链接校验失败的原因有哪些，如何处理？

A: 链接校验失败可能源于网络超时、目标服务器拒绝连接、HTTP 4xx/5xx 状态码或 SSL 证书问题。首先检查本地网络环境，确认可正常访问外网。若为特定链接反复失败，可在 `data/resources.json` 中将该条目的 `status` 字段标记为 `unreachable` 并记录时间，随后通过人工访问确认链接是否永久失效。若链接有效但校验脚本误报，可调整 `scripts/check_links.py` 中的超时参数或 User-Agent 头。

Q: 如何批量导入已有的书签文件？

A: ResourceBridge 提供了 `scripts/import_bookmarks.py` 辅助脚本，支持从 Netscape 格式的 HTML 书签导出文件（常见于 Chrome 和 Firefox）以及 JSON 格式的浏览器书签备份进行导入。执行 `python scripts/import_bookmarks.py --input bookmarks.html --format netscape` 即可将书签中的链接自动归入资源索引，原书签的文件夹层级将被映射为分类标签。

Q: 本地服务启动后看不到任何资源条目？

A: 请确认 `data/resources.json` 文件存在且内容格式合法。初次启动需运行 `python scripts/init_db.py --import data/resources.json` 完成数据初始化。若文件缺失，可从 `data/resources.sample.json` 复制一份作为起始数据集。另外，检查 `scripts/serve.py` 的运行日志，确保无 JSON 解析错误或文件读取权限异常。
