# LinkPivot Core

LinkPivot Core is a high-performance technical resource aggregation and navigation system designed for developers, researchers, and technical writers who need to organize, categorize, and reference large volumes of external article links. The project addresses the fundamental challenge of managing distributed knowledge across heterogeneous content sources by providing a structured, queryable, and extensible link catalog with built-in validation and metadata enrichment pipelines.

Target users include open-source maintainers building documentation portals, data engineers curating training corpora, and technical archivists preserving reference materials. Unlike generic bookmark managers, LinkPivot Core operates at the repository level, treating links as first-class assets with version-controlled manifest files, automated accessibility checks, and batch processing capabilities for bulk link operations. The system supports both static site generation and API-driven consumption models.

## 功能概览

**批量链接摄入与规范化** - 支持从文本文件、CSV 和直接粘贴的原始列表中批量摄入 URL，自动执行协议归一化、冗余去重和格式清洗。

**存活与可达性探测** - 内置异步 HTTP 健康检查器，可配置超时与重试策略，对每个链接执行 GET 请求并记录状态码、响应时间和内容哈希。

**元数据自动提取** - 从目标页面解析标题、描述、关键词和主要正文摘要，生成可用于索引和搜索的结构化元数据。

**多级标签分类系统** - 支持用户自定义标签体系，可为链接分配多个分类标签，并提供标签树视图与统计仪表板。

**静态站点生成器** - 将链接目录渲染为纯 HTML 文档树，支持按标签、时间、域名聚合，生成适合托管在 GitHub Pages 或任何静态服务器上的导航站点。

**RESTful API 服务** - 提供 JSON-over-HTTP 接口，支持链接查询、筛选、随机抽取和批量导出，便于集成到外部工作流。

**变更审计日志** - 记录所有链接的添加、删除、元数据更新和状态变更事件，支持时间旅行回溯和差异对比。

**配置化抓取规则** - 通过 YAML 配置文件定义针对不同域名的请求头、延迟策略和内容解析选择器，适应多样化的目标站点结构。

## 应用场景

**技术文档站外链管理** - 大型开源项目维护数千篇外部参考文章链接，LinkPivot Core 可自动检测死链并生成失效报告，确保文档引用质量。维护者每月运行一次全量检查即可获得待修复链接清单。

**研究文献数据整理** - 学术团队在系统综述或元分析阶段需要整理数百篇网络文章，系统提供标签筛选和批量元数据导出功能，可直接生成 BibTeX 或 CSV 格式的参考文献草稿。

**企业内部知识仓库构建** - 技术中台团队将分散在各业务组的分享文档、故障复盘报告和技术博客聚合为统一入口，利用标签分类实现跨团队知识检索，并定时抓取更新状态。

**个人开发学习笔记** - 独立开发者将技术文章按语言、框架、难度分级归档，配合静态站点生成器快速搭建个人学习知识库，实现本地 Markdown 笔记与在线链接资源的双向关联。

## 快速开始

```bash
# 克隆项目仓库
git clone https://github.com/linkpivot/linkpivot-core.git
cd linkpivot-core

# 安装依赖（使用虚拟环境）
python3 -m venv venv
source venv/bin/activate  # Windows 下使用 venv\Scripts\activate
pip install -r requirements.txt

# 使用示例数据运行摄入流程
python -m linkpivot ingest --input samples/links.txt --output data/manifest.json
python -m linkpivot check --manifest data/manifest.json --workers 10
python -m linkpivot build --manifest data/manifest.json --output dist/
```

## 贡献指南

## 常见问题

问：系统如何处理目标站点反爬机制？

系统提供可配置的请求延迟、随机 User-Agent 轮换和 Cookie 持久化支持。用户可在 config.yaml 中按域名指定延迟区间（单位秒），并启用 IP 轮换代理（需外部代理池）。对于严格限制的站点，建议将并发工作线程数（--workers）设置为 1 并增加延迟，或手动将此类链接标记为需人工验证。

问：静态站点生成后的页面样式是否可以自定义？

所有 HTML 模板位于 src/linkpivot/exporter/templates/ 目录，使用 Jinja2 语法。用户可直接修改模板文件或覆盖默认样式表（style.css）。生成命令支持 --template-dir 参数，允许指定外部模板目录，便于在不修改源码的情况下进行完全自定义。

问：如何从其他链接管理工具迁移数据？

系统内置了转换脚本位于 scripts/ 目录，目前支持 Raindrop.io 的 JSON 导出格式和 Pocket 的 HTML 导出格式。对于其他格式，用户可将链接列表整理为纯文本（每行一个 URL）或 CSV（列：url,title,tags），然后使用 ingest 命令的 --format 参数指定对应格式。
