# ResourceBridge

ResourceBridge 是一个面向技术开发者与架构师的高质量外链资源聚合与导航系统。项目定位于对分散于网络各处的技术文档、教程、案例分析与工程实践文章进行结构化收录与分类索引，解决开发者在信息检索过程中面临的海量噪音、链接失效与内容质量参差等问题。本项目不对原始内容进行转载或修改，仅提供稳定、可校验的入口地址与元信息描述，适用于个人知识管理、团队技术选型参考以及自动化文档爬虫的种子源。

## 功能概览

**自动化链接健康检查**：系统周期性对收录的全部外链发起可用性探测，记录响应状态码与响应时间，自动标记异常链接。

**多维度分类与标签体系**：每个资源条目支持归属多个技术领域、难度等级与内容类型，便于按场景快速过滤。

**全文元信息提取**：对目标页面进行标题、发布时间、作者、章节结构等基础元数据抽取，生成简要摘要供预览。

**外部数据导入导出**：支持批量导入用户自定义链接列表，同时提供 JSON、CSV、Markdown 表格等多种格式的导出接口。

**链接变更追踪**：记录每个资源页面的历史标题与摘要变更，帮助用户感知内容演化趋势。

**访问热度统计**：基于内部代理日志计算各资源的被访问频次与平均停留时长，辅助判断内容实用度。

**自定义分类视图**：允许用户创建个人收藏夹与分类标签，将公共资源库映射到个人工作空间。

**API 查询接口**：提供 RESTful API 供外部系统按关键词、分类、时间范围等条件进行远程检索。

## 应用场景

技术团队内部知识库建设：团队可将 ResourceBridge 部署为内部文档导航入口，集中管理常用的框架官方文档、最佳实践文章与故障排查案例，减少重复搜索成本。

自动化文档爬虫的种子源管理：数据采集系统可将本项目的资源列表作为初始 URL 集合，配合元信息字段进行增量爬取与内容去重。

个人开发者学习路线规划：开发者可按分类标签浏览资源，快速定位某一技术栈的入门教程、进阶分析与实战案例，形成系统性学习路径。

技术选型调研辅助：架构师在进行中间件或框架选型时，可通过本平台检索对应的性能对比报告、迁移经验与生产环境落地总结。

## 快速开始

以下步骤指导您在本地环境中快速启动 ResourceBridge 服务。

```bash
# 克隆代码仓库
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge

# 安装项目依赖（使用 pip 与虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化数据库并导入示例资源
python manage.py migrate
python manage.py loaddata seed_resources.json

# 启动开发服务器
python manage.py runserver 0.0.0.0:8000
```

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 及以上 | 核心运行时环境，提供解释器与标准库 |
| Django | 4.2 LTS | Web 框架，用于路由、ORM 与管理界面 |
| PostgreSQL | 14 及以上 | 主数据库，存储资源条目、分类与变更记录 |
| Redis | 7.0 及以上 | 缓存与任务队列后端，用于链接健康检查异步任务 |
| Celery | 5.3 及以上 | 分布式任务队列，编排周期性探测与统计聚合 |
| Gunicorn | 21.0 及以上 | WSGI HTTP 服务器，用于生产环境部署 |
| Nginx | 1.24 及以上 | 反向代理与静态文件服务，提供访问控制与负载均衡 |
| nodejs | 18 及以上 | 前端构建工具链依赖，用于编译静态资源 |
| yarn | 1.22 及以上 | JavaScript 包管理器，用于前端依赖锁定与构建 |

## 常见问题

Q: 系统支持的最大资源条目数量是多少？是否存在硬性上限？

A: 系统本身不设硬性上限。在 PostgreSQL 配合合适索引的情况下，实测可稳定管理五十万条资源。当条目超过十万时，建议开启 Redis 缓存并配置分页查询，同时将健康检查任务间隔调整为 24 小时以上以降低数据库压力。

Q: 如何自定义链接健康检查的超时时间与重试策略？

A: 可在 src/apps/resources/services/checker.py 中调整 HTTP_TIMEOUT 与 RETRY_COUNT 变量。如需动态配置，可通过 Django Admin 界面修改 SiteConfig 表中的对应键值，系统会在下一次周期任务时自动读取新配置。

Q: 导入外部链接列表时，是否支持自动去重？

A: 导入流程默认基于 URL 的完全匹配进行去重。若需忽略查询参数或路径末尾斜杠，可在导入时启用 normalize_url 选项，该选项会对 URL 进行标准化处理后再执行去重比较。
