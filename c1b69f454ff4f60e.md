# TechLink Navigator

TechLink Navigator 是一个面向技术研究人员、开发者与开源爱好者的结构化外链资源聚合平台。本项目并不直接存储任何文章内容，而是通过对特定领域内高质量外部链接进行系统性归类、索引与版本追踪，帮助用户快速定位到与移动端技术、后端架构、运维实践及语言生态相关的深度资料。项目定位为“技术资源的引航员”，尤其适用于需要批量查阅特定站点下技术文章批次的场景，例如跟踪某一站点下的历史技术文档、进行批量链接可用性检测，或是构建自定义的技术阅读清单。

本项目由一批长期关注移动互联网技术演进的开发者维护，当前批次收录了 180 条来自 `mobile.occzl.cn` 及其子域名的技术文章链接。这批链接覆盖了从基础编程语法到高阶系统设计的广泛主题，经过结构化整理后可无缝集成至各类自动化工作流中，包括但不限于链接监控、内容聚合与知识图谱构建。

## 功能概览

**批量链接导入与解析** 支持一次性导入大规模外链列表，自动识别 URL 结构并提取文章标识符，便于后续元数据采集。

**多维度标签分类** 根据链接来源域名、路径层级及文章 ID 区间进行自动打标，用户可自定义分类规则，实现个性化资源视图。

**链接可用性健康检查** 内置异步 HTTP 探活机制，周期性检测列表中每个资源的可达性，自动标记失效链接并生成报告。

**结构化数据导出** 支持将整理后的资源列表导出为 JSON、CSV 或纯文本格式，方便导入至其他数据分析工具或文档系统。

**版本差异对比** 针对不同批次（如当前第 28/56 批）的资源列表提供差异比对功能，清晰展示新增、删除及变更的链接。

**自定义元数据扩展** 允许用户为每条链接附加自定义标签、阅读状态、优先级及备注信息，构建个人知识库。

**命令行交互界面** 提供轻量级 CLI 工具，支持通过终端执行链接查询、分类筛选与统计操作，满足自动化脚本调用需求。

**Web 可视化看板** 基于简单静态页面展示资源总览、分类占比及健康状态，方便团队内部共享资源池概览。

## 应用场景

**技术文档历史归档查询** 当团队需要追溯某一技术主题在特定站点上的历史发布脉络时，可通过本项目快速定位到对应批次的链接集合，结合文章 ID 规律进行时间序列分析。

**自动化链接监控运维** 运维人员可配置本项目为定时任务，定期扫描资源列表中的每个 URL，当检测到大量链接失效时及时发出告警，避免文档系统中出现死链碎片。

**技术周报素材采集** 内容编辑者可利用本项目的分类筛选功能，从数百条链接中按主题快速筛选出符合当周技术焦点的高质量文章，作为技术周报的外部引用来源。

**知识图谱关系构建** 数据工程师可将导出的结构化链接数据与其它内部知识库进行关联，通过文章 ID 与域名信息建立外部资源与内部文档的映射关系，丰富知识图谱的边信息。

## 快速开始

以下步骤将指导您在本地环境中快速启动 TechLink Navigator 的核心服务。

```bash
# 1. 克隆项目仓库至本地
git clone https://github.com/techlink-navigator/tln-core.git
cd tln-core

# 2. 安装项目依赖（使用 Python 虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 3. 导入当前批次资源列表并执行初始健康检查
python tln.py import --batch 28 --source resources/batch_28.txt
python tln.py check --batch 28 --concurrency 50 --output report_batch_28.json
```

## 项目结构

项目采用分层架构设计，核心模块与辅助工具分离，便于维护与扩展。

```
tln-core/
├── src/                                # 源代码主目录
│   ├── tln/                            # 核心包
│   │   ├── __init__.py                 # 包初始化，暴露主要接口
│   │   ├── cli.py                      # 命令行入口，定义子命令组
│   │   ├── importer.py                 # 链接导入模块，支持 txt/csv 格式解析
│   │   ├── checker.py                  # 异步链接检查器，基于 aiohttp 实现
│   │   ├── classifier.py               # 分类引擎，加载 yaml 规则并匹配
│   │   ├── exporter.py                 # 导出模块，支持 json/csv/xlsx 格式
│   │   └── models.py                   # 数据模型定义（Link, Batch, Report）
│   └── utils/                          # 通用工具函数
│       ├── logger.py                   # 日志配置与封装
│       ├── validators.py               # URL 格式校验与规范化辅助
│       └── file_utils.py               # 文件读写与路径处理
├── tests/                              # 测试目录
│   ├── unit/                           # 单元测试，针对各模块独立功能
│   │   ├── test_importer.py
│   │   ├── test_checker.py
│   │   └── test_classifier.py
│   └── integration/                    # 集成测试，验证端到端流程
│       └── test_batch_flow.py
├── config/                             # 配置文件目录
│   ├── default.yaml                    # 默认分类规则与检查参数
│   └── custom_rules/                   # 用户自定义规则存放位置
│       └── example_rule.yaml
├── resources/                          # 资源数据目录
│   ├── batch_28.txt                    # 第 28 批次原始链接列表（180 条）
│   ├── batch_28_metadata.json          # 批次元数据（导入时间、来源等）
│   └── reports/                        # 检查报告输出目录
│       └── report_batch_28_20260702.json
├── docs/                               # 项目文档源码
│   ├── user_guide.md
│   ├── batch_management.md
```

## 常见问题

**问：导入链接时提示“URL 格式不合法”，但我在浏览器中可以直接打开该链接，原因是什么？**

答：本项目在导入阶段会对 URL 进行严格的结构校验，要求必须包含协议头（http:// 或 https://）。您提供的原始数据中若存在类似 `www.mobile.occzl.cn` 这样缺少协议头的字符串，校验器会拒绝导入。解决方案是在导入前使用文本编辑器的替换功能，为所有缺失协议头的条目统一添加 `http://` 前缀。本项目当前版本（v1.0）暂不支持自动补全协议头，以避免误补全非 URL 字段。

**问：并发检查时出现大量超时或连接重置错误，如何优化？**

答：此类错误通常由网络环境或目标站点的访问频率限制引起。建议您采取以下措施：首先，降低并发数，可在 `check` 命令中通过 `--concurrency` 参数设置为 20 或更低；其次，增加单次请求的超时时间，默认值为 10 秒，可调整为 30 秒；最后，检查本地网络是否存在代理或防火墙限制，必要时将本项目所在机器加入出口白名单。若问题持续存在，可将检查模式切换为串行（`--serial` 参数）以确保稳定性，但会显著增加总耗时。

**问：导出的 Excel 文件中部分链接显示为超链接格式，如何取消？**

答：在导出模块中，我们默认将 URL 列设置为纯文本格式以避免 Excel 自动转换。若您发现仍然出现可点击的超链接，可能是由于您的 Excel 版本开启了自动识别超链接功能。您可以在打开文件后，选中 URL 列，通过“数据”选项卡下的“分列”功能，选择“文本”格式重新转换一次。或者在系统设置中关闭 Excel 的“自动更正”中的“Internet 及网络路径替换为超链接”选项。
