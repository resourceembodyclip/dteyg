# LinkHarbor

LinkHarbor 是一个面向技术团队与内容研究者的结构化外链资源汇聚平台。该项目不生产原始内容，而是通过严格的链接分类、状态监控与元数据标注机制，将分散于互联网各处的优质技术文章、工具文档与案例解析整合为可检索、可追溯、可共享的知识索引体系。

项目定位为技术资源导航中间件，服务于需要快速定位特定技术主题原始资料的一线开发人员、技术决策者与开源贡献者。LinkHarbor 不对下游链接做内容摘要改写，仅提供直达原始出处的高保真访问路径，确保信息传递的零损耗与可验证性。

## 功能概览

**按原始域名聚合分组**：自动识别并归类全部收录链接的根域名，支持按来源站点的维度批量导出链接清单，便于团队评估特定信源的整体价值。

**URL 原样保留机制**：系统强制保留用户提交的每一个 URL 的协议、大小写、路径后缀与文件名扩展名，杜绝任何形式的自动补全或规范化改写，保证链接指向的确定性。

**链接可达性定时巡检**：每日对资源池内全部链接执行 HEAD 请求与状态码检查，标记异常链接并生成告警日志，辅助维护人员及时清理或替换失效条目。

**自定义标签与全文备注**：允许用户为每条链接附加技术领域标签、内容类型标签与自由文本备注，支持后续按标签组合筛选与导出。

**批量导入与增量更新**：提供基于纯文本列表或 CSV 格式的批量 URL 导入接口，支持按批次号管理资源集合并记录每次更新的时间戳与操作人。

**只读访问与镜像隔离**：平台本身不缓存、不代理、不转储下游页面内容，所有外链均以原始协议直接访问，避免版权争议与内容同步延迟问题。

**结构化导出接口**：支持将全部或筛选后的链接列表导出为 JSON、YAML 与纯文本三种格式，便于集成至上游知识管理工具或自动化流水线。

## 应用场景

技术团队内部知识库建设。团队可将日常阅读中积累的技术博客、官方文档与疑难问题解答链接统一录入 LinkHarbor，通过标签系统按微服务、数据库、前端框架等维度分类，形成团队共享的技术外链知识树。

开源项目文档外链维护。开源项目维护者可使用 LinkHarbor 管理 README、Wiki 或官网中引用的所有外部参考链接，借助巡检功能及时发现第三方文档的迁移或失效情况，避免用户因坏链产生困惑。

技术调研与竞品分析。在进行技术选型或竞品分析时，调研人员可将收集到的竞品官方公告、技术白皮书、社区讨论帖等链接批量导入系统，通过批次号与备注字段记录调研日期与初步判断，形成可回溯的调研证据链。

个人技术阅读工作流管理。开发者可将每日浏览中发现的值得细读的文章链接存入 LinkHarbor，利用备注功能记录阅读优先级或待深入研究的要点，避免信息过载导致的遗忘与重复查找。

## 快速开始

以下命令序列演示了从代码仓库克隆、安装依赖到启动本地服务的完整流程。

```bash
git clone https://github.com/linkharbor/linkharbor.git
cd linkharbor
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver 0.0.0.0:8000
```

执行完毕后，访问 localhost:8000 即可进入管理界面。首次启动将自动创建默认管理员账户，账户信息在控制台日志中输出，请及时修改初始密码。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.10 及以上 | 核心运行环境，低于此版本将无法解析类型注解语法 |
| PostgreSQL | 14.0 及以上 | 主数据库，用于存储链接元数据、标签与批次信息 |
| Redis | 6.2 及以上 | 缓存与任务队列后端，支撑定时巡检任务调度 |
| Node.js | 18.0 及以上 | 仅用于前端静态资源构建，生产环境可仅安装运行时 |
| Git | 2.30 及以上 | 版本控制工具，用于克隆仓库与获取更新 |
| pip | 22.0 及以上 | Python 包依赖管理工具，用于安装 requirements 中的三方库 |

│   └── celery.py             # Celery 应用实例定义，用于配置定时任务与队列
├── apps/                     # 功能应用模块集合
│   ├── links/                # 链接管理核心应用
│   │   ├── models.py         # 定义 Link、Tag、Batch 等数据模型及字段约束

│   │   ├── serializers.py    # DRF 序列化器，定义 API 输入输出的数据结构
│   │   └── tasks.py          # Celery 定时任务，执行链接可达性巡检与异常记录
│   ├── users/                # 用户与权限管理应用

提交问题报告前，请先查阅已有 Issue 列表避免重复。新建 Issue 时需明确标注问题类型（缺陷、功能请求或文档改进），并提供可复现步骤或充分的使用场景描述。

代码贡献需遵循 PEP 8 编码规范，并确保新增或修改的代码包含对应的单元测试用例，测试覆盖率不得低于现有平均水平。提交前请运行全部测试套件确保无回归错误。

文档类贡献欢迎补充更多使用案例、故障排查指南或最佳实践分享，文档存放于 docs 目录下，采用标准 Markdown 格式书写，中文表述需保持专业、简洁、无歧义。

## 常见问题

问：巡检功能检测到链接异常后，系统会自动修复或替换链接吗？

答：系统仅负责检测并记录异常状态，不会自动修改或替换任何用户提交的链接内容。异常链接列表会在管理后台的巡检日志模块中高亮显示，用户需人工确认链接是否永久失效，并手动决定保留、移除或更新为新地址。

问：批量导入链接时，是否会对重复 URL 进行去重处理？

答：系统会基于链接的完整字符串进行精确去重。若导入列表中包含已存在于数据库中的完全相同 URL，则新条目将被忽略并记录在导入日志中。但请注意，系统不会对协议不同或大小写不同的相似链接做模糊去重，用户导入前应先自行做标准化处理。

问：项目是否会对下游链接做内容快照或本地缓存？

答：LinkHarbor 设计为纯外链导航系统，不对任何下游页面的内容、图片或附件进行抓取、缓存或代理转发。所有访问请求均由用户浏览器直接向原始服务器发起，平台本身不存储任何第三方内容，以此避免版权风险并减轻服务器存储压力。
